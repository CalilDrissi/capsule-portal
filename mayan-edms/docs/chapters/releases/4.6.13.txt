Version 4.6.13
==============

Released: March 7, 2026


Summary
-------

This maintenance release focuses on forward compatibility, feature and improvement backports, and safer dependency handling. It also includes broad dependency refreshes, Docker image updates, and several improvements to settings, security defaults, and large-file processing.

Notable highlights include:

- Upload wizards now skip cabinet and tag steps when the user has no access.
- Forms and settings framework backports to improve forward compatibility.
- Hardened outbound HTTP requests with sensible timeout defaults.
- New chunked hashing helper optimized for large files.
- Dependency and Docker image updates across the stack.
- Series support update: 4.6 is now End-Of-Life and upgrades are required.


Changes
-------

Backward incompatible changes
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

- The setting ``VIEWS_SHOW_DROPZONE_SUBMIT_BUTTON`` was renamed to ``FORMS_SHOW_DROPZONE_SUBMIT_BUTTON``.
- The default Gunicorn worker class changed from ``sync`` to ``gevent``. This can affect concurrency behavior and operational tuning.
- Older ``aarch64`` and ``armv7l`` compatibility code was removed.


UI / UX
^^^^^^^

- Updated the cabinet and tag upload wizard steps to match the metadata wizard permission logic. When a user has access to none of the relevant objects, the step is skipped entirely instead of being shown without usable options.
- Added a thousand separator to the pager for improved readability.
- Updated the password reset flow to navigate to the login URL instead of the logout URL.
- Improved settings view return navigation and fixed a repeated return link in the setting edit view.


Forms
^^^^^

- Backported the ``forms`` app to improve forward compatibility.
- Moved the DropZone widget implementation from views into the forms app.
- Removed hard-coded DropZone icons.


Settings
^^^^^^^^

- Backported the settings class split and duplicate namespace detection.
- Added a test to ensure all setting namespaces render correctly.
- Backported settings framework updates:

  - Split class modules.
  - Split test modules.
  - Improved setting and namespace deregistration behavior.

- Updated ``SEARCH_MATCH_ALL_DEFAULT_VALUE``:

  - The default value is now a boolean.
  - The setting is now a choice field for clearer configuration.

- Added and refined help text for settings:

  - ``DOCUMENTS_DISPLAY_HEIGHT``
  - ``DOCUMENTS_DISPLAY_WIDTH``
  - ``DOCUMENTS_PREVIEW_HEIGHT``
  - ``DOCUMENTS_PREVIEW_WIDTH``
  - ``DOCUMENTS_PRINT_HEIGHT``
  - ``DOCUMENTS_PRINT_WIDTH``
  - ``SIGNATURES_BACKEND_ARGUMENTS``
  - ``STORAGE_DOWNLOAD_FILE_STORAGE_ARGUMENTS``

- Added many missing ``message=`` keyword arguments.
- Simplified and clarified multiple help text strings.


Security and networking defaults
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

- Hardened outbound HTTP requests by introducing default timeout behavior:

  - Dependency downloads now use 10 seconds for initial connection and 30 seconds for initial data transfer.
  - Added the ``oidc_discovery_timeout`` keyword argument to ``AuthenticationBackendOIDC``. Defaults to 5 seconds for connection and 15 seconds for response data.

- Switched JavaScript package downloads to use the TLS version of NPM.


Performance and reliability
^^^^^^^^^^^^^^^^^^^^^^^^^^^

- Added a reusable chunked hashing function optimized for large files.
- Set the default chunked hashing block size to 65536 bytes to better align with typical allocation and buffering characteristics.
- Reinforced pruning of empty index instance nodes.
- Reduced logging output in several areas.
- Switched to an explicit iterator chunk size for more predictable batching.


Dependency and library management
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

- Modernized ``PythonDependency`` version checking by removing reliance on the deprecated ``pkg_resources`` library.
- Updated the version check implementation to use the newer PyPI API.
- Added keyword arguments support to ``sh.Command`` where still used.
- Added a new management command ``dependencies_uninstall`` to remove JavaScript and Google Font dependencies.
- Updated ``django-formtools`` from 2.3 to 2.5.1, and added a workaround for an undocumented backward-incompatible issue introduced in ``django-formtools`` 2.4.0.
- Replaced ``sh`` usage with Python's upstream ``subprocess`` for:

  - ``MIMETypeBackendFileCommand``
  - ``MIMETypeBackendPerlFileMIMEInfo``

- Added ``libfuse2`` as a production dependency.
- Added ``pkg-config`` as a build-time dependency.

Documentation and localization
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

- Updated translation-related code.
- Updated documentation to use a general-purpose environment variable loader.
- Added guidance to help users avoid breaking installations when attempting to bypass localization.


Workflow and internal refactors
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

- Moved the model ``WorkflowTransitionField`` to its own module.
- Removed dependency on Django's ``make_random_password`` method.
- Removed dependency on Django ``utils.timezone``.
- Refactored ``ContentTypeCheckTestCaseMixin``.

^^^^^^^^^^^^^^^^^

- Updated the series support chart. Series 4.6 is now End-Of-Life. Series 4.6 was supported from January 2024 through March 2026 (2 years and 2 months). Users must upgrade to at least series 4.7.
