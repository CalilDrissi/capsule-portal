Version 4.6.14
==============

Released: April 16, 2026


Summary
-------

This maintenance release tightens security around templating and form widgets, continues internal refactoring work in templating and metadata, and includes routine dependency and Docker base image updates.


Changes
-------

Templating
^^^^^^^^^^

- Split the templating template tags module to improve maintainability.
- Prefixed templating tag and filter functions to reduce the chance of name collisions with reserved words and other template utilities.
- Backported support for marking template tags and filters as "dangerous". These are tags or filters that, if exposed without proper controls, can grant more access than intended. This change introduces the setting
  `TEMPLATING_TAGS_DANGEROUS_ALLOW_LIST`, which defaults to allowing the existing tags for backward compatibility.
- Backported deterministic assurance logic for AJAX template rendering.


Metadata
^^^^^^^^

- Split metadata models to reduce coupling and support clearer separation of responsibilities.
- Ensure a metadata type parser and validator are always available.


Documents and sources
^^^^^^^^^^^^^^^^^^^^^

- Improved `SourceStoredFile` filename decoding, increasing robustness when handling encoded and non-ASCII filenames.


Indexing
^^^^^^^^

- Cleaned up and hardened document indexing widgets.


Forms and widgets
^^^^^^^^^^^^^^^^^

- Hardened the `PlainWidget` form widget and removed usage of `mark_safe` to reduce the risk of unsafe HTML rendering.
- Continued general hardening and cleanup of indexing related widget code.


Dependency & library management
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

- Updated Django from version 4.2.29 to 4.2.30.


Docker
^^^^^^

- Updated the base Docker image from `debian:12.13-slim` to `debian:bookworm-20260406-slim`.
